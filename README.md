# 餐厅收银系统 (Restaurant POS System)

基于 Vue 3 + Element Plus + IndexedDB 的纯前端餐厅收银系统，无需后端服务。

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3 | ^3.4 | 前端框架 (Composition API) |
| Vite | ^5.4 | 构建工具 |
| Element Plus | ^2.7 | UI 组件库 |
| @element-plus/icons-vue | ^2.3 | 图标库 |
| Vue Router | ^4.3 | 路由管理 |
| ECharts | ^5.5 | 图表库 |
| vue-echarts | ^7.0 | Vue 3 ECharts 封装 |
| IndexedDB | 原生 | 浏览器端数据持久化 |

## 功能清单

### 1. 登录认证

- **路由**: `/login`
- **账号**: `admin` / `123456`
- 登录状态存储在 `localStorage`，路由守卫拦截未登录请求
- 登录页不显示侧边栏，登录后自动跳转 POS 收银页

### 2. POS 收银

- **路由**: `/`
- 左侧菜品区：按分类筛选（全部 / 热菜 / 凉菜 / 主食 / 饮品 / 甜品），网格卡片展示
- 点击菜品卡片添加到购物车，已添加的菜品在卡片右上角显示数量角标
- 右侧购物车：展示已选菜品列表，支持数量加减、单项删除、一键清空
- 底部合计金额与商品数量

#### 现金支付

- 弹出对话框输入收款金额
- 提供快捷金额按钮（自动向上取整至 10 的倍数）
- 自动计算找零金额
- 收款不足时禁止确认

#### POS 支付

- 模拟 POS 刷卡流程
- 确认后显示"POS机处理中..."（2 秒模拟延迟）
- 处理完成后提示支付成功

### 3. 菜品管理

- **路由**: `/menu`
- 表格展示所有菜品：ID、名称、分类标签、价格
- 支持按分类筛选
- **添加菜品**: 弹出表单，支持从已有分类选择或输入新分类
- **编辑菜品**: 点击编辑按钮修改名称、分类、价格
- **删除菜品**: 二次确认后删除
- 数据实时同步至 IndexedDB

### 4. 订单历史

- **路由**: `/orders`
- 展示所有历史订单，默认按时间倒序
- 筛选条件：日期范围（起止日期）、支付方式（现金 / POS 机）
- 每笔订单显示：订单号、商品明细（名称 × 数量 + 小计）、总金额、支付方式、收款/找零、交易时间
- 顶部汇总：当前筛选条件下的订单数和总销售额

### 5. 销售报表

- **路由**: `/reports`
- 四项统计图表：
  - **近 7 天销售额趋势**: 折线图，带面积填充
  - **各分类销售占比**: 环形饼图，按菜品分类汇总销售额
  - **支付方式占比**: 环形饼图，按现金 / POS 机汇总订单数
  - **菜品销量排行 Top 10**: 横向柱状图，按销售份数排名
- 底部汇总卡片：总订单数、总销售额、平均客单价

## 数据模型

### dishes 表（菜品）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number (自增) | 主键 |
| name | string | 菜品名称 |
| category | string | 所属分类 |
| price | number | 单价（元） |

### orders 表（订单）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number (自增) | 主键 |
| items | array | 订单明细快照（含 name, price, category, qty） |
| totalAmount | number | 订单总金额 |
| paymentMethod | string | 支付方式：现金 / POS机 |
| cashReceived | number | 实收现金（仅现金支付） |
| change | number | 找零金额（仅现金支付） |
| createdAt | string (ISO 8601) | 交易时间 |

## 版本历史

| 版本 | Git Tag | 说明 |
|------|---------|------|
| v1.0 | `v1.0-desktop` | 桌面版：侧边栏导航 + el-table 表格布局 |
| v2.0 | `v2.0-tablet` | 平板适配版：底部 TabBar 导航 + 响应式卡片布局 + 模拟小票打印 |

### v2.0 变更摘要

- **导航**：左侧侧边栏 → 底部 TabBar（更适合平板触控操作）
- **POS 收银**：菜品区 + 购物车双栏，窄屏自动竖向堆叠
- **菜品管理**：el-table → 响应式卡片行布局
- **订单历史**：el-table → 卡片列表布局
- **销售报表**：图表在平板竖屏下自动改为单列
- **小票打印**：已预留模拟打印功能（代码中 `模拟打印` 标记区域），接入实体打印机时替换为蓝牙 ESC/POS 指令即可
- **登录页**：卡片宽度自适应

回滚到桌面版：`git checkout v1.0-desktop`

## 浏览器兼容性

需要支持 IndexedDB 的现代浏览器（Chrome 50+、Firefox 44+、Edge 79+、Safari 10+）。

## 注意事项

- 所有数据存储在浏览器 IndexedDB 中，清除浏览器数据会导致数据丢失
- 订单中的菜品信息以交易时的快照保存，后续修改菜品不影响历史订单
- 登录状态存储在 `localStorage`，退出登录或清除 `localStorage` 会回到登录页
